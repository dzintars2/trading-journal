# Forex Trading Journal App - Design Requirements ## Core Functionality - **Trade Logging:** Capture entry price, exit price, stop loss, take profit, position size, and timestamps. - **Visual Documentation:** Capability to upload and store multiple screenshots for each trade (e.g., higher timeframe context, entry setup, exit reason). - **Performance Tracking:** Automated calculation of R-multiple, P/L (in pips and currency), and win rate. - **Dashboard:** Visual representation of trading data using charts and key performance indicators (KPIs). ## User Flows 1. **Adding a Trade:** User enters trade details, uploads screenshots of the setup, and saves the entry. 2. **Closing a Trade:** User updates an open trade with exit data and final screenshots. 3. **Reviewing Performance:** User visits the dashboard to see growth over time and identify strengths/weaknesses. 4. **Browsing History:** User filters and searches through the trade log to find specific past setups. ## Design Aesthetic - **Professional & Clean:** Data-heavy but organized to prevent cognitive overload. - **High Contrast:** Clear distinction between winning (green) and losing (red) trades. - **Dark/Light Mode:** Support for both, with a focus on a professional "trading terminal" feel. - **Desktop First:** Optimized for large screens and multi-tasking.
